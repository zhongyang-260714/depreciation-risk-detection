---

## 一、项目简介

AI 技术迭代速度远快于会计准则更新周期，导致科技企业资产折旧存在严重错配：
- **时间错配**：技术迭代周期(1-3年) << 会计摊销周期(5-10年)
- **规则错配**：旧准则(工业时代) → 新现实(数字时代，无形资产主导)
- **激励错配**：管理层美化业绩动机 + 审计师技术理解不足

本项目构建**AI驱动的科创企业资产折旧风险识别系统**，通过多模态数据融合与可解释AI，实现财报风险精准识别与预警。

## 二、技术架构（五层）

```
┌──────────────────────────────────────────────────────────┐
│  应用层：投资者决策 / 审计师辅助 / 监管预警              │
├──────────────────────────────────────────────────────────┤
│  输出层：风险评分(0-100) / 风险类型 / 预警报告 / 仪表盘   │
├──────────────────────────────────────────────────────────┤
│  AI模型层：                                               │
│  • XGBoost/LightGBM 风险分类                             │
│  • 孤立森林 / AutoEncoder 异常检测                        │
│  • BERT/FinBERT 文本风险抽取                             │
│  • LLM 风险报告自动生成                                  │
├──────────────────────────────────────────────────────────┤
│  特征工程层：                                             │
│  • 传统财务指标（折旧率、无形资产占比）                    │
│  • 科创特有风险指标（研发资本化率、技术半衰期）            │
│  • 文本特征（年报关键词情感分析）                          │
├──────────────────────────────────────────────────────────┤
│  数据输入层：                                             │
│  • 财务报表（CSMAR / WRDS / Compustat）                   │
│  • 专利数据（USPTO / Google Patents）                    │
│  • 年报文本（SEC EDGAR / 巨潮资讯）                       │
│  • 外部数据（宏观经济、行业政策）                          │
└──────────────────────────────────────────────────────────┘
```

## 五、快速开始

### 1. 环境安装

```bash
# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate    # Windows

# 安装依赖
pip install -r requirements.txt
```

### 2. 数据准备

详见 `docs/data_guide.md`，数据目录（`data/`）需手动下载，不在版本控制内。

### 3. 运行示例

```bash
# 特征工程 + 模型训练（示例）
python -m src.models.train --config config/default.yaml

# 启动Streamlit演示界面
streamlit run src/dashboard/app.py
```
