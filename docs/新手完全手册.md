### 1.3 安装 Git（第一次使用必须做）

Git 是一个把代码从电脑上传到 GitHub 的工具。你只需要装一次。

步骤 1：打开浏览器，访问 `https://git-scm.com/download/win`

### 1.4 配置 Git 用户名和邮箱（只做一次）

在刚才打开的 Git Bash 黑色窗口里，依次输入下面两条命令，每输完一条按回车：

```bash
git config --global user.name "zhongyang-260714"
git config --global user.email "你的邮箱@example.com"
```

> 把 `你的邮箱@example.com` 换成你注册 GitHub 时用的真实邮箱。

### 3.2 创建虚拟环境（每个项目做一次）

虚拟环境的作用是隔离不同项目的依赖，防止冲突。

步骤 1：打开命令提示符（cmd），输入：

```cmd
cd C:\Users\Lenovo\Documents\kimi\workspace\depreciation-risk-detection
python -m venv venv
```

步骤 2：激活虚拟环境：

```cmd
venv\Scripts\activate
```

如果成功，命令行前面会出现 `(venv)` 字样，像这样：
```
(venv) C:\Users\Lenovo\...\depreciation-risk-detection>
```

> ⚠️ 每次你要运行项目代码，都要先执行 `venv\Scripts\activate`

## 第六部分：常见问题（FAQ）

### Q1：GitHub 上传时提示"Authentication failed"
A：GitHub 从 2021 年起不再支持密码登录。你需要：
1. 访问 GitHub → 右上角头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 点击 Generate new token，勾选 `repo` 权限
3. 复制生成的 token，在 Git Bash 里弹窗要求密码时，输入这个 token 而不是你的密码

### Q2：飞书知识库里的文档怎么设置权限？
A：飞书知识库默认只有群成员能访问。如果需要更细的控制，点击文档右上角「分享」→ 设置权限。

### Q3：运行 `pip install` 时很慢或报错？
A：使用国内镜像源：
```cmd
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q4：Streamlit 运行后浏览器打不开？
A：检查命令行里显示的地址。如果显示 `localhost:8501`，用浏览器打开 `http://localhost:8501`。如果显示其他端口，用那个地址。

### Q5：队友改了代码，我想更新到最新版本？
A：在 Git Bash 里执行：
```bash
git pull origin main
```

### Q6：我不小心把重要文件删除了，能恢复吗？
A：如果已经提交到 GitHub 了，可以恢复。告诉我具体情况，我帮你处理。
