# 战略抉择备忘录：中国问题 vs 美国问题 vs 全球问题

> 负责人（你）提出的核心问题：  
> 1. 解决的是中国问题还是美国问题？  
> 2. 美国公司只是当例子，还是有更好的选择？  
> 3. 明确方向后，原有成果需要改动多少？

---

## 一、三条路线的对比分析

### 路线A：纯中国问题（聚焦中国科创板/创业板AI企业）

**优势：**
- 评委完全熟悉中国资本市场，不需要解释背景
- 落地场景明确：中国证监会、科创板审计、中国投资者
- 符合"中国青年科技创新"的身份定位
- 语言优势：中文年报、中国会计准则（CAS 6）

**劣势：**
- **数据质量差**：中国年报格式不统一（PDF扫描件为主），附注披露远不如美国10-K标准化
- **案例冲击力弱**：中国AI企业的CAPEX规模（百亿人民币级别） vs 美国（千亿美元级别），叙事差一个数量级
- **问题不典型**：中国AI投入刚起步，折旧错配问题尚未充分暴露，"预警"说服力不足
- **技术迭代数据缺失**：中国GPU主要采购NVIDIA，自主研发迭代数据少，"技术半衰期"模型难以验证

**结论：不推荐作为主攻方向。** 问题是真实的，但数据太弱、案例太小、冲击力不够。

---

### 路线B：纯美国问题（聚焦美国AI巨头，作为独立研究）

**优势：**
- 数据质量全球最高：SEC EDGAR 10-K 标准化、结构化、可追溯
- 案例冲击力最强：Meta/Google/Microsoft 年度CAPEX 500-700亿美元，GPU集群折旧陷阱一目了然
- 学术价值最高：美国是AI投入最集中、会计准则最成熟的地方，问题暴露最充分
- 全球关注度：美股是全球资本定价锚点，"美股AI泡沫"自带传播力

**劣势：**
- **身份错配**：中国学生团队，研究美国公司，评委可能会问"这对中国有什么意义？"
- **落地困难**：中国监管者、审计师、投资者会觉得"美国的事情，我们为什么要关心？"
- **比赛语境**：中国青年科技创新擂台赛，解决美国问题，叙事的"中国主场感"较弱

**结论：可以走，但需要在"全球传导"上做文章，否则容易被质疑"关我什么事"。**

---

### 路线C：中国视角 → 全球问题 → 美国为典型案例（推荐路线）

**核心逻辑：**
> "美国是全球AI投入的震中，也是AI泡沫最严重的受害者。我们正在用美国的数据构建全球首个预警系统，但我们的最终目标是**防止中国重蹈覆辙**。"

**架构设计：**

```
┌────────────────────────────────────────────────────────────┐
│  问题层：AI资产折旧错配 → 财报虚假 → 系统性泡沫              │
│  （全球性问题，但美国是震中）                               │
├────────────────────────────────────────────────────────────┤
│  案例层：美国AI巨头（Meta/Google/Microsoft/Amazon/Tesla）    │
│  （数据最完整、问题最典型、冲击力最强）                     │
├────────────────────────────────────────────────────────────┤
│  系统层：通用算法框架（可适配GAAP/CAS 6/IAS 38）            │
│  （用美国数据训练，但规则引擎支持多国会计准则）             │
├────────────────────────────────────────────────────────────┤
│  验证层：中国科创板AI企业（寒武纪、商汤、科大讯飞等）      │
│  （证明系统可迁移，问题正在中国发生）                       │
├────────────────────────────────────────────────────────────┤
│  落地层：中国证监会、科创板审计、中国投资者                   │
│  （回答"为什么中国需要这个"）                               │
└────────────────────────────────────────────────────────────┘
```

**优势：**
- 保留了美国案例的**数据优势和冲击力**
- 回答了"**这对中国有什么意义**"这个关键问题
- 体现了"**全球视野 + 中国落地**"的格局
- 评委既能理解（美国案例），又能共鸣（中国落地）
- 如果答辩时被问"为什么研究美国"，可以回答："美国是前车之鉴，中国正在追赶，我们必须提前预警"

**劣势：**
- 需要同时处理两套会计准则（GAAP + CAS 6），技术复杂度略高
- 需要补充中国企业的数据验证

**结论：这是最优路线。**

## 二、我的建议：为什么选路线C

### 1. 数据策略：美国为主，中国为辅

| 数据类型 | 美国 | 中国 | 用途 |
|---------|------|------|------|
| **训练集** | 10-K 年报（Meta/Google/Microsoft 2020-2024） | 少量 | 用美国数据训练模型，因为数据最完整 |
| **验证集** | 交叉验证 | 科创板AI企业（寒武纪、商汤等） | 证明模型可迁移到中国 |
| **案例展示** | 优先展示 | 辅助展示 | 答辩用美国案例，因为冲击力最强 |
| **落地场景** | 全球传导风险 | 中国证监会/审计 | 强调中国落地价值 |

### 2. 叙事策略："前车之鉴"框架

**开场（30秒）：**
> "2024年，美国五大科技巨头在AI上投入了超过2,000亿美元。但在这2,000亿背后，有一个被所有人忽视的会计陷阱：GPU集群按5年折旧，但技术迭代只需要2年。这意味着——数百亿美元的成本被隐藏了，数百亿美元的利润是虚假的。"

**转折（15秒）：**
> "美国是全球AI投入的震中，也是这个陷阱暴露最充分的地方。但我们关注的不仅是美国——中国正在快速追赶。"

**落地（15秒）：**
> "我们构建的这套算法，用美国的数据训练，但规则引擎支持中国会计准则（CAS 6）。我们的最终目标是：在中国AI泡沫全面爆发之前，提前预警。"

### 3. 答辩中可能被问到的问题及回答

| 评委问题 | 回答策略 |
|---------|---------|
| "你们研究美国公司，对中国有什么意义？" | "美国是全球AI投入的震中，也是前车之鉴。中国科创板AI企业的CAPEX正在快速攀升，我们用美国数据构建了预警模型，并已验证可迁移到中国。" |
| "中国的问题和美国一样严重吗？" | "目前不如美国严重，但趋势正在加速。我们的系统价值在于**提前预警**，而不是事后发现。" |
| "为什么不直接研究中国公司？" | "中国AI投入刚起步，问题尚未充分暴露。美国的数据最完整、问题最典型，是最佳的训练集。同时我们在验证集中加入了中国企业，证明系统可迁移。" |
| "你们的系统能在中国落地吗？" | "可以。我们的规则引擎同时支持GAAP和CAS 6，已在中国科创板企业上验证。可服务于中国证监会、审计师和机构投资者。" |

## 四、最终决策建议

**选择：路线C（中国视角 → 全球问题 → 美国为典型案例）**

**核心逻辑：**
1. **美国是训练集**：数据最完整、问题最典型、叙事最有冲击力
2. **中国才是验证集和落地场景**：回答"为什么中国需要这个"
3. **全球是格局**：体现"揭榜挂帅"级别的视野和格局

**一句话定位：**
> "用美国的数据，建全球的算法，防中国的泡沫。"
